# N.I.W.O.E.C.
Now your project works on everyone's computer, test your app through virtual environments, ensuring it works one everyone's computer.

================================================================================
                        N.I.W.O.E.C.
   NOW IT WORKS ON EVERYONE'S COMPUTER
================================================================================

A local, AI-powered hardware simulator that tells you whether your application 
will run on a given hardware configuration. 

- Enter your target CPU, RAM, and GPU VRAM
- If the target exceeds your real hardware, the app forces an AI analysis
- Provide a requirements.txt describing your app
- The AI predicts "Works" or "Doesn't work" with a confidence score

Everything runs entirely on your machine – no cloud, no servers.

================================================================================
                        SYSTEM REQUIREMENTS
================================================================================
- Rust (stable) – https://rustup.rs
- Python 3.8+  (only needed to generate the AI model files once)
- Operating System: Windows, Linux, or macOS

================================================================================
                        DOWNLOAD & EXTRACT
================================================================================
1. Clone or download this repository.
2. You should have a folder named "niwoec" with the following structure:

   niwoec/
   ├── Cargo.toml
   ├── Cargo.lock
   ├── src/
   │   └── main.rs
   ├── models/                <-- you will create this
   │   ├── model.onnx         <-- generated by Python script
   │   └── tokenizer.json     <-- generated by Python script
   ├── train_model.py         <-- AI training script
   └── README.txt             <-- this file

================================================================================
                        STEP 1: GENERATE THE AI MODEL FILES
================================================================================
The AI model is not included in the repository due to its size.
You must run the provided Python script **once** to create it.

1. Open a terminal / command prompt inside the "niwoec" folder.
2. Install Python dependencies:

   pip install torch transformers onnx

3. Run the training script:

   python train_model.py

   This will download a small transformer model (~23 MB) and create:
   - models/model.onnx
   - models/tokenizer.json
   - (and a few other tokenizer helper files)

   The training uses dummy data, so predictions will be random until you
   replace the data in train_model.py with your own real app results.

================================================================================
                        STEP 2: BUILD THE RUST APPLICATION
================================================================================
1. Make sure you have Rust installed (rustc --version).
2. From the "niwoec" folder, run:

   cargo build --release

3. The first build will take several minutes (it compiles ONNX Runtime).
4. After it finishes, the executable is at:

   target/release/niwoec       (Linux / macOS)
   target/release/niwoec.exe   (Windows)

================================================================================
                        STEP 3: RUN THE APPLICATION
================================================================================
Double-click the executable, or run it from the terminal:

   ./target/release/niwoec      (Linux / macOS)
   target\release\niwoec.exe    (Windows)

The dark-themed window will open.

**Quick usage guide:**

   ┌─────────────────────────────────────────────────┐
   │ Your PC Specifications                         │
   │   CPU: 8 cores @ 3600 MHz    RAM: 16384 MB     │
   │   GPU VRAM (manually adjustable): [4096]       │
   ├─────────────────────────────────────────────────┤
   │ Target Hardware Configuration                  │
   │   CPU Cores: [ 8]                              │
   │   RAM (MB):  [16384]                           │
   │   GPU VRAM (MB): [4096]                        │
   ├─────────────────────────────────────────────────┤
   │ ✔ Target is within your hardware capabilities  │
   │ [Simulate (AI optional)]   [🤖 AI Analyze]     │
   ├─────────────────────────────────────────────────┤
   │ Result                                         │
   │   No analysis run yet.                         │
   └─────────────────────────────────────────────────┘

   - Adjust "GPU VRAM (manually adjustable)" to match your actual GPU memory.
   - Change the target specs to the hardware you want to test.
   - If the target is higher than your host, only the "AI Analyze" button
     will be active (the simulation button disappears).
   - Click "AI Analyze" and choose your requirements.txt file.
   - The AI will show "✅ Works" or "❌ Doesn't work" with a confidence %.

================================================================================
               ABOUT THE requirements.txt FILE
================================================================================
The AI expects a plain text file describing your application's requirements.
Write in natural language – the AI extracts meaning from it.

Example (requirements.txt):
   App needs 60 FPS, real-time physics, 4K textures, multiplayer support.

The more detailed the description, the better the AI can assess compatibility.

================================================================================
                IF YOU WANT TO RETRAIN THE AI WITH REAL DATA
================================================================================
Open train_model.py and replace the dummy lists:

   texts = [...]       # app requirement descriptions
   hardware = [...]    # [cpu_cores, ram_mb, gpu_vram_mb]
   labels = [...]      # 1 = works, 0 = doesn't work

Add as many real examples as you can (dozens or hundreds). Then re-run:

   python train_model.py

The new model will be saved to models/ and the Rust app will use it immediately.

================================================================================
                        TROUBLESHOOTING
================================================================================
Q: When I run the Rust app, it says "Failed to load model.onnx".
A: Make sure the "models" folder is **next to the executable**. If you moved
   the .exe file, copy the whole "models" folder with it.

Q: "tokenizers" or "ort" crate fails to compile.
A: Try using stable ort 1.15. In Cargo.toml change:
       ort = "1.15"
   and in src/main.rs change the AiModel::new() to use the older API
   (see comments in the code).

Q: GPU VRAM detection doesn't work.
A: Automatic GPU detection is not implemented. Use the manual slider to set
   your GPU's VRAM (check Task Manager / System Profiler).

Q: The AI gives random answers.
A: The default model is trained on dummy data. Train it with real examples
   to get meaningful predictions.

================================================================================
                        BUILDING A PORTABLE VERSION
================================================================================
After building, you can zip the following files together:

   niwoec.exe   (or niwoec)
   models/
       model.onnx
       tokenizer.json

Anyone can run the executable on the same OS without installing Rust or Python.
